# Buy or Bye Design System

> 이 문서는 분석 웹 UI의 디자인 기준이다. 앞으로 `analyzer/style.css`, `analyzer/index.html`, `analyzer/app.js`에서 UI를 수정할 때는 이 문서를 먼저 기준으로 삼는다.

## 1. 디자인 방향

- 전체 인상은 **세련된 대시보드형 구매 판단 도구**다.
- 시각적 분위기는 **정돈됨, 신뢰감, 빠른 스캔 가능성, 실사용성**을 우선한다.
- 컬러는 Step마다 다르게 나누지 않고, **단일 dashboard accent**를 중심으로 통일한다.
- 강한 색은 CTA, 현재 활성 상태, 중요한 판단에만 사용한다.
- 유리감, 과한 블러, 큰 배경 그라데이션, 장식용 그림자는 지양한다.

## 2. 레이아웃 원칙

- 좌측 기능 레일은 유지한다. 주요 기능 진입점이며, 화면 전체 레이아웃의 기준점이다.
- 본문은 레일을 제외한 영역에서 중앙 정렬된 dashboard canvas로 배치한다.
- 한 화면에서 정보가 길어질 때는 카드가 겹치지 않고 세로 흐름으로 자연스럽게 이어져야 한다.
- Step 영역은 각각 독립된 surface처럼 보이되, 과한 배경 박스를 만들지 않는다.
- 빈 공간은 기능적 여백이어야 한다. 내용이 없는 큰 공백, 카드 내부의 강제 높이, 버튼 주변의 과한 padding은 피한다.
- 텍스트가 잘리면 안 된다. 제목만 필요한 경우를 제외하면 `nowrap`, line clamp, fixed height를 신중히 사용한다.

## 3. 컬러 토큰

기본 색상은 아래 의도를 따른다. 실제 CSS 변수명은 기존 코드와 호환되게 유지해도 되지만, 의미는 이 기준에 맞춘다.

| 용도 | 권장 색 | 사용 위치 |
|---|---:|---|
| App background | `#f5f7fb` 또는 유사 cool gray | body, app shell |
| Surface | `#ffffff` | 카드, 패널, 입력창 |
| Soft surface | `#f0f3ff` / `#f7f8fc` | 강조 카드, 안내 카드 |
| Text primary | `#20242c` | 제목, 핵심 값 |
| Text secondary | `#697282` | 설명, 보조 정보 |
| Line | `#e6e9f1` | 카드 테두리 |
| Mint action | `#14b8a6` | 공통 액션 버튼, 다시 만들기, Step 5 action |
| Mint hover | `#0f8f82` | 액션 버튼 hover |
| Step accent set | Orange/Blue/Purple/Rose/Teal | Step별 라벨과 배경 tint |

## 4. Step 컬러

Step 영역은 강한 색상으로 구분하지 않고, 가벼운 glass-like surface로 통일한다. 성능 이슈가 있었으므로 `backdrop-filter` 대신 투명 배경, 얇은 라인, 낮은 그림자로만 glass 느낌을 만든다.

| 용도 | 색상 | 사용 규칙 |
|---|---:|---|
| Step surface | translucent white `rgba(255,255,255,0.56~0.72)` | 모든 Step 동일 |
| Step label | no background + strong text shadow | 배경 박스 없이 텍스트만 또렷하게 |
| Action buttons | Mint `#14b8a6` | 모든 액션 버튼 동일 |

## 5. 카드 스타일

- 기본 카드:
  - 배경: 흰색
  - 테두리: 얇은 neutral line
  - radius: 14-18px
  - shadow: 작고 은은하게
- 시작 카드/다음 단계 CTA:
  - 기본 상태는 단색 당근 오렌지 + 흰 텍스트를 사용한다.
  - hover 시 단색 당근 오렌지로 바뀌고 텍스트 대비는 유지한다.
  - 그라데이션은 사용하지 않는다.
- 정보 카드:
  - 긴 문장은 자연스럽게 줄바꿈한다.
  - fixed height보다 content-driven height를 우선한다.

## 6. 버튼 원칙

- 기본 액션 버튼은 Step과 무관하게 Step 5 민트 `#14b8a6` + 흰 텍스트로 통일한다.
- hover에서는 더 진한 민트 `#0f8f82`로 바꾸고 텍스트는 계속 흰색이다.
- 다시 만들기, 다시 검색, 외부 검색, 이전/다음 단계 버튼은 모두 같은 민트 버튼 스타일을 쓴다.
- 보조 버튼은 흰 배경 + 얇은 테두리 + 어두운 텍스트.
- 버튼은 눌리지 않았는데 active처럼 보이면 안 된다.
- 아이콘 버튼은 배경을 최소화하고, hover/focus에서만 surface를 준다.

## 7. 좌측 레일

- 좌측 레일은 앱의 기능 네비게이션으로 유지한다.
- 레일은 본문과 분리된 고정 영역이다.
- active 상태는 실제 열린 패널이나 현재 상태에만 유지한다.
- 단발성 버튼(다시 분석, 새 분석 등)은 클릭 후 active 배경이 남지 않게 한다.
- 레일 색은 현재 톤을 유지하되, 본문 디자인보다 시각적으로 과하게 튀지 않게 한다.

## 7.1 Step 3 가격 참고

- 관련 매물은 가로 카드 그리드가 아니라 세로 리스트로 보여준다.
- Step 3는 주황 사용을 줄이고 흰색/중립 회색 surface 중심으로 구성한다.
- 표, 검색어 칩, 안내 문구는 같은 neutral spacing과 radius를 사용한다.

## 8. Step 4 영수증

- 영수증 출력 중 종이는 가로로 잘리면 안 된다.
- 영수증이 아래로 출력되면 Step 4 배경도 같은 속도로 자연스럽게 늘어나야 한다.
- Step 5 진입 카드는 영수증 출력 완료 후, Step 4 배경 아래에 나타난다.
- 영수증은 종이 느낌을 유지하되 주변 배경과 겹쳐 보이지 않아야 한다.
- 영수증 재출력은 텍스트 버튼 대신 회전 화살표 아이콘 버튼을 사용한다.

## 9. Step 5 판매자 대화

- Step 5는 채팅 도구처럼 보여야 한다.
- 채팅 영역은 충분히 읽기 쉬워야 하지만, 빈 공간이 과하면 안 된다.
- 말투 버튼, 입력창, 추천 칩은 compact하게 유지한다.
- 판매자 답변 입력창은 대화 흐름 아래에 자연스럽게 위치한다.
- 추천 메시지 클릭/전송 시 전체 스레드가 리렌더된 느낌을 주지 않고 새 메시지만 추가되는 느낌이어야 한다.

## 10. 금지 사항

- 새 디자인을 위해 임의의 대형 배경 그라데이션을 추가하지 않는다.
- 유리감(`backdrop-filter`)은 성능 문제가 있었으므로 사용하지 않는다.
- `!important`를 남발해 장기적으로 cascade를 더 꼬이게 하지 않는다. 불가피하면 파일 끝에 이유 있는 작은 override로 제한한다.
- 특정 화면을 고치면서 다른 step의 레이아웃 원칙을 깨지 않는다.
- active, hover, selected 상태를 시각적으로 혼동시키지 않는다.

## 11. 작업 체크리스트

UI를 수정할 때마다 아래를 확인한다.

- 이 변경이 이 문서의 컬러/간격/상태 규칙과 맞는가?
- hover 전 기본 상태가 너무 튀지 않는가?
- 텍스트가 잘리지 않는가?
- Step 4 영수증 출력과 Step 5 등장 순서가 유지되는가?
- 좌측 레일 기능이 유지되는가?
- slide/scroll 모드가 모두 정상인가?
