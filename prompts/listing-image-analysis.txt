역할:
중고 매물 사진 {{imageCount}}장을 순서대로 확인하고, 각 사진에 대해 짧은 구매 판단 코멘트를 작성하세요.

입력:
제품명: {{productName}}
제목: {{title}}
본문 참고:
{{body}}

사진 해상도 (첨부 이미지와 동일 순서):
{{sizeLines}}

처리 기준:
좌표 방식:
- 각 사진은 원본 이미지와, 같은 사진을 감싸는 큰 배경 위에 25열(A-Y) × 25행(1-25) 좌표 격자 보드로 합성한 비교 이미지가 함께 제공됩니다.
- 그리드 보드에는 바깥 테두리 라벨뿐 아니라 이미지 안쪽에도 열 문자와 행 숫자 안내 라벨이 반복 표시됩니다. 좌표를 셀 때는 하자 가까이에 있는 내부 라벨과 우측/하단 외곽 라벨을 함께 확인하세요.
- 하자 여부는 원본 이미지에서 먼저 확인하고, 하자 위치와 크기는 반드시 그리드 이미지 기준으로 산출하세요.
- defects[].gridCenter는 25×25 그리드 기준으로 하자 정중앙이 들어가는 셀 하나입니다. 예: "M16".
- defects[].gridSizeCells는 하자를 감싸는 정수 셀 블럭의 크기입니다: {"cols": 가로 칸 수, "rows": 세로 칸 수}. 최소값은 반드시 5×5입니다. 점상 오염, 작은 찍힘, 작은 부품 누락도 {"cols":5,"rows":5} 이상으로 잡고, 가로 기스는 cols를 크게, 세로 기스는 rows를 크게 잡으세요.
- gridCenter와 gridSizeCells는 별개입니다. 하자가 외곽 셀에 있으면 gridCenter는 외곽 셀 그대로 두고, gridSizeCells만 5×5 이상으로 키우세요. 블럭을 이미지 안에 맞추려고 gridCenter를 W18처럼 안쪽 정상 영역으로 옮기면 안 됩니다.
- 작은 부품 누락, 찍힘, 점상 오염처럼 작은 하자는 특히 실제 하자 한가운데에 가장 가까운 gridCenter를 고르세요. 하자 범위의 좌상단 셀을 gridCenter로 쓰면 안 됩니다.
- bbox/center/size 퍼센트 좌표는 쓰지 마세요. 서버가 gridCenter와 gridSizeCells를 실제 표시 좌표로 변환합니다.
- 좌표 선택은 설명보다 중요합니다. defects에 넣기 전 반드시 "내가 고른 gridCenter 셀이 실제 하자 픽셀 위에 겹치는가?"를 한 번 검증하세요. 셀이 정상 부품이나 정상 표면 위에 있으면 틀린 좌표입니다.
- defects[].gridReason에는 왜 그 셀을 골랐는지 1문장으로 쓰세요. 예: "손상 중심이 X17 부근에 있어 선택".
1. 원본 이미지에서 실제로 보이는 하자 픽셀/빈 부분/깨진 경계/색이 끊긴 부분을 먼저 찾습니다. 제목·본문 설명은 무엇을 찾아볼지 참고만 하고, 좌표는 반드시 이미지에서 보이는 픽셀 기준으로 정합니다.
2. 그리드 이미지에서 그 실제 하자 픽셀이 들어있는 셀들을 찾습니다. 주변의 비슷한 정상 물체, 글자, 아이콘, 반복 패턴, 밝은 반사처럼 눈에 잘 띄는 다른 대상에 끌려가면 안 됩니다.
3. gridCenter는 "하자 설명에 해당하는 실제 픽셀 덩어리의 중심이 들어가는 셀"입니다. 하자 근처의 정상 부품 중심, 빈 공간이 아닌 정상 표면, 또는 하자와 다른 물체 위의 셀을 선택하지 마세요.
4. gridSizeCells는 gridCenter가 포함된 정수 셀 블럭으로, 실제 하자 픽셀을 전부 감싼 뒤 사방으로 약간의 여유를 둡니다. 어떤 하자든 5×5보다 작게 출력하지 마세요.
5. 하자가 두 행/두 열 사이 또는 외곽에 걸쳐 있으면, 실제 하자 중심에 가장 가까운 셀을 gridCenter로 고르고 gridSizeCells를 5×5, 5×6, 6×5처럼 키워 인접 셀까지 포함하세요.
6. 최종 출력 전에 gridCenter 셀과 gridSizeCells 블럭 안에 description의 하자가 실제로 들어있는지 다시 확인하세요. 들어있지 않으면 그 defect는 좌표를 다시 고르거나 출력하지 마세요.
- images[].index는 위에 적힌 사진 번호를 그대로 쓰세요. 묶음 안 순번이 아니라 전체 매물 사진 번호입니다.
- 가장 먼저 "이 사진이 중고 판매자가 직접 찍은 실물 근거인가?"를 판별하세요. 이 판별이 하자 판단보다 우선입니다.
- 사진 안에 가격, 원가 취소선, 할인율, 쿠폰, 쿠폰보기, 장바구니, 구매하기, 리뷰, 별점, 상품명/가격 카드, 쇼핑몰 상세 UI가 보이면 실제 상품이 함께 보여도 반드시 홍보용·쇼핑몰 스크랩 이미지로 분류하세요.
- 위 경우에는 label을 "홍보 이미지" 또는 "실물 확인 불가"로 쓰고, level은 "caution"으로 쓰세요.
- 홍보용·쇼핑몰 스크랩 이미지에서는 "깨끗해 보입니다", "양호합니다", "오염이나 파손이 없습니다", "구성품이 잘 갖춰져 있습니다", "본문 설명과 일치합니다"처럼 실물 상태를 확인한 듯한 표현을 절대 쓰지 마세요.
- 홍보용·쇼핑몰 스크랩 이미지의 comment는 반드시 "쇼핑몰/홍보용 캡처로 보여 이 사진만으로는 실제 박스·본체·구성품 상태를 확인할 수 없습니다."와 같은 취지로 작성하세요.
- 판매자가 직접 찍은 실물 사진의 흔적이 있으면 홍보용으로 분류하지 마세요. 예: 방/책상/바닥/침구/포장재/택배박스/스티로폼/비닐/케이블/충전기/구성품이 실제 공간에 놓여 있음, 조명·초점·그림자·손떨림·압축감이 있음, 박스 내부 구성품을 촬영함, 제품이 포장 트레이나 완충재 안에 들어 있음.
- 특히 박스 내부 구성품 사진, 언박싱처럼 보이는 사진, 판매자가 보유 물품을 한 장에 펼쳐 찍은 사진은 깨끗하고 정돈되어 보여도 실물 근거로 보세요. 이런 경우 label은 "구성품 확인", "박스 상태", "실물 구성"처럼 쓰고, 실제로 보이는 범위에서만 상태를 판단하세요.
- 가격/쿠폰 UI가 없더라도 공식 렌더·카탈로그·패키지 아트처럼 보일 수는 있지만, 단순히 사진이 깨끗하거나 중앙 정렬되어 있다는 이유만으로 홍보용으로 단정하지 마세요. 실제 배경·포장재·구성품·촬영 흔적이 전혀 없고 공식 이미지처럼 제품만 완벽하게 분리되어 있거나 본문 맥락과 맞지 않을 때만 홍보용·스크랩 가능성을 의심하세요.
- 판매자가 직접 찍은 실물 사진으로 명확할 때만 찍힘/스크래치/오염/구성품 누락/박스 상태를 구체적으로 판단하세요.
- 구성품 확인 사진이면 label을 "구성품 확인"처럼 쓰고, 보이는 구성품만 comment에 적으세요. 기준 구성품 목록이 주어지면 사진에서 실제로 보이는지 우선 확인하세요.
- 각 이미지별로 "문제 없어 보입니다", "모서리 찍힘이 의심됩니다", "구성품 확인이 필요합니다"처럼 짧고 구체적으로 작성합니다.
- 확실하지 않으면 단정하지 말고 "의심", "확인 필요"라고 씁니다.
- 사진에서 보이는 하자/오염/찍힘/스크래치/구성품 누락 가능성/박스 상태를 우선 봅니다.
- 실물 사진으로 판단되면 먼저 이미지 전체를 네 구역(좌상/우상/좌하/우하)으로 훑고, 모서리·테두리·힌지·포트·액정 가장자리·버튼 주변·박스 모서리처럼 실제 중고 하자가 자주 생기는 부위를 우선 확인하세요.
- 하자 후보를 찾을 때는 원본 이미지에서 색/질감/형태가 주변과 끊기는 부분을 확인한 뒤, 그리드 이미지에서 해당 하자의 중심 셀과 차지하는 칸 수를 계산하세요.
- 직접적으로 하자·오염·찍힘·스크래치·깨짐·찢김·도장 까짐이 의심되는 위치가 보일 때만 images[].defects에 적으세요.
- 여러 비슷한 물체나 부품이 주변에 있으면, description에 쓴 하자와 같은 실제 대상 위에 gridCenter가 놓였는지 다시 비교하세요. 하자 설명과 좌표가 서로 다른 대상을 가리키면 defects에 넣지 말고 다시 고르세요.
- 이미지 안에 같은 모양의 부품이나 반복 패턴이 여러 개 있으면, "가장 눈에 띄는 비슷한 대상"이 아니라 실제 하자 픽셀이 있는 대상의 셀을 선택하세요.
- 하자 위치가 사진 가장자리나 구석에 있으면, 가장자리 밖 여백이나 가까운 정상 물체가 아니라 실제 보이는 하자 영역의 중심 셀을 선택하세요. 하자가 이미지 외곽에 붙어 있으면 A열/Y열/1행/25행 같은 외곽 좌표를 그대로 써야 합니다. 하자가 우측 끝 Y18~Y19 부근에 걸쳐 있으면 gridCenter는 "Y18" 또는 "Y19"처럼 외곽 셀로 두고 gridSizeCells를 {"cols":5,"rows":5}처럼 넉넉하게 잡으세요.
- 화면에 표시된 앱 아이콘·배경 이미지·잠금화면 내용·조명 반사·유리 반짝임·손/카메라 그림자·압축 노이즈·먼지는 실제 표면 손상과 구분하세요. 이런 요소만 보이면 defects에 넣지 말고 comment에서 확인 필요로만 언급하세요.
- 선형 흠집은 실제 표면 위에 고정된 밝거나 어두운 선인지 확인하고, 단순 반사광처럼 가장자리가 흐리거나 화면 내용과 함께 움직일 법한 무늬라면 defects에 넣지 마세요.
- defects[].gridCenter는 25×25 그리드 이미지 기준으로 하자·기스·흠집의 정중앙이 들어가는 셀입니다. 하자 범위의 좌상단이 아니라 중심 셀 하나만 쓰세요.
- defects[].gridSizeCells는 gridCenter가 포함된 상태로 그 하자를 감싸는 정수 셀 블럭 크기입니다. 실제 표시 원이 너무 작아지지 않도록, 작은 하자도 기본적으로 5×5칸을 쓰고 하자가 선형이면 방향에 맞춰 더 넓게 잡으세요.
- 긴 균열·넓은 액정 파손·여러 줄 스크래치처럼 넓게 퍼진 하자는 중심을 실제 손상 묶음의 가운데 셀에 두고, gridSizeCells를 실제 손상이 걸친 그리드 칸 수만큼 키우세요.
- 마지막으로 각 defect마다 gridCenter 셀, gridSizeCells 범위, description이 같은 실제 하자 하나를 가리키는지 자체 검수하세요.
- 예시: 작은 점상 오염이 M16 부근이면 {"gridCenter":"M16","gridSizeCells":{"cols":5,"rows":5}}, 세로 기스가 T5-T12에 걸치면 {"gridCenter":"T9","gridSizeCells":{"cols":5,"rows":8}}, 가로 스크래치가 H7-N9에 걸치면 {"gridCenter":"K8","gridSizeCells":{"cols":7,"rows":5}}, 우측 끝 외곽 하자가 Y18~Y19에 붙어 있으면 {"gridCenter":"Y18","gridSizeCells":{"cols":5,"rows":5}} 또는 {"gridCenter":"Y19","gridSizeCells":{"cols":5,"rows":5}}처럼 외곽 셀을 그대로 쓰세요.
- defects[].description은 "우측 상단 미세 스크래치 의심"처럼 짧게 쓰세요.
- defects[].approximateSize는 보이면 "약 1cm", "작은 점상", "세로 2~3cm"처럼 대략 크기를 쓰고, 모르겠으면 빈 문자열로 두세요.
- 정상으로 보이는 부분, 그림자/반사/먼지처럼 하자인지 애매한 부분은 defects에 넣지 마세요. comment에만 "확인 필요"라고 쓰세요.
- 사진에서만 알 수 있는 "근거"를 comment에 포함하세요. 예: 구성품이 실제로 보이는지, 박스/케이스 모서리 상태, 광택·오염·흠집 의심, 본문과 사진의 불일치 가능성.
- label은 사진 위 배지로 표시할 짧은 AI 요약명입니다. 사진 내용을 보고 4~10자 한국어로 직접 만드세요. 예: "전면 상태", "구성품 확인", "흠집 의심", "작동 근거", "박스 상태", "사진 부족", "홍보 이미지", "실물 확인 불가", "주의 사진".
- 같은 성격의 사진은 반드시 같은 label을 쓰세요. 예: 측면을 보여주는 사진 여러 장은 모두 "측면 상태", 전면을 보여주는 사진은 "전면 상태", 박스는 "박스 상태"처럼 통일합니다.
- level은 실물 사진에서 상태가 양호하면 safe, 추가 확인이 필요하거나 홍보용·스크랩 이미지라 실물 상태를 알 수 없으면 caution, 실제 하자·불일치가 강하게 보이면 risk, 판단 근거가 거의 없으면 neutral로 쓰세요.
- 이미지 분류 블록은 서버가 images[].label 기준으로 자동 생성합니다. groups는 빈 배열 [] 로 두세요.
- overall에는 홍보용·스크랩 이미지가 많아 실물 근거가 부족한 경우 그 점을 직접 요약하세요.
- 토큰 절약을 위해 comment는 사진당 1문장으로 쓰세요.

출력 형식:
반드시 JSON만 출력하세요.

출력 스키마:
{
  "images": [
    {
      "index": 1,
      "width": 1200,
      "height": 900,
      "label": "전면 상태",
      "comment": "본체 전면이 실제 사진으로 보여 큰 흠집은 뚜렷하지 않지만 모서리 상태는 추가 확인이 필요합니다.",
      "level": "caution",
      "defects": [
        { "gridCenter": "M16", "gridSizeCells": { "cols": 2, "rows": 1 }, "gridCell": "M16", "gridReason": "스크래치 중심이 M16 셀 위에 걸쳐 있어 선택", "description": "우측 상단 미세 스크래치 의심", "approximateSize": "작은 선형", "severity": "caution" }
      ]
    }
  ],
  "groups": [
  ],
  "overall": "사진 전체 기준 요약 1문장"
}
