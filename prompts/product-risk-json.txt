{
  "relatedIssues": [
    { "title": "검색된 실제 이슈명", "detail": "중고 구매자가 왜 신경 써야 하는지 구체적으로", "level": "safe|caution|risk" }
  ],
  "chronicDefects": [
    { "title": "구체적 증상·부품·상태명", "detail": "구매 전 확인 이유와 실제로 어떤 문제가 생기는지", "level": "safe|caution|risk" }
  ],
  "verdict": "추천/주의/비추천 중 하나와 이유 한 문장"
}

규칙:
- 조사 메모에 나온 내용만 사용하세요.
- relatedIssues는 검색 근거가 있는 제품 관련 이슈·논란·주의점만 최대 3개.
- chronicDefects는 반복 고장, 내구성 약점, 반복적으로 확인되는 구매 리스크만 최대 3개. 해당 제품에 물리적 고질병이나 반복 리스크가 뚜렷하지 않으면 빈 배열 []로 둡니다.
- title에는 "최근 이슈", "고질병", "AS·유통" 같은 범주명 대신 구체적인 증상·부품·상태를 쓰세요.
- level은 구매 영향이 작으면 safe, 확인 필요면 caution, 수리비·사용성에 큰 영향이면 risk.
- 판매자에게 물어볼 질문은 만들지 마세요.
- "정보 부족", "검색했습니다", "AI 기능" 같은 메타 문구는 쓰지 마세요.
- 조사 메모가 "근거 없음", "일반론 제외", "뚜렷한 이슈 없음"이라고 판단한 내용을 부정적인 카드로 바꾸지 마세요.
- "공식 제품 불확실", "비공식 제품 가능성", "가짜 제품 가능성", "사기 가능성"은 카드로 만들지 마세요.
