역할:
조사 메모를 분석하여 중고 구매 리스크를 JSON으로 요약하십시오.

입력:
제품명: {{productName}}
조사 메모: {{researchText}}

처리 기준:
조사 메모의 내용을 아래 필드에 맞게 배치합니다. 막연한 사기·진위 의심은 쓰지 않고, 모델 전용 근거가 아니면 detail에 "제품군 기준" 등 범위를 밝히세요.

relatedIssues — 관련 이슈(최소 2개, 빈 배열 금지)
- 중고로 이 제품·이 매물을 살 때 직접 확인할 주의점입니다.
- 호환성, 구성품, 계정/락, 사용 조건, 판매글 누락 등 제품·매물 단위 이슈를 넣습니다.
- 모델 고질병이 부족하면 판매글·제품군 확인 포인트로 채웁니다.

chronicDefects — 고질병·반복 리스크(최소 1개)
- 같은 모델·제품군에서 반복되는 고장, 내구 약점, 소모·교체 부담입니다.
- 이어패드·배터리·필터 교체 비용·주기, 특정 부위 파손 취약점은 여기에 둡니다.

marketFactors — 시장 변수(최대 1개, 없으면 [])
- 동일 모델·제품군의 중고가 밴드가, 이 매물 상태와 무관한 시장 전체 이유로 움직일 때만 넣습니다.
- 판단: 「이 매물의 하자·소모품·수리」가 아니라 「같은 제품 중고 시장」 자체가 바뀌는 요인인지 구분합니다.
- 해당 시: 단종·재고, 후속작 출시, 범용 부품·램·SSD 시세, 환율, AS·공식 유통 종료, 수요·공급 변화.
- 작성: 같은 제품군 중고가가 왜 변하는지 한 문장으로 설명합니다. 근거가 뚜렷하지 않으면 []로 둡니다.

purchaseChecklist — 판매자 확인(최대 2개)
- 거래 전 판매자에게 바로 물을 질문·요청입니다. 이 매물에 필요한 것만 넣습니다.

공통
- level: safe | caution | risk
- title은 짧은 명사구, detail·verdict는 존댓말 완결 문장
- "정보 부족" 등 메타 문구 금지

출력 형식:
JSON 객체 하나만 반환. 설명/코드펜스 금지.

출력 스키마:
{
  "relatedIssues": [
    {
      "title": "이슈명",
      "detail": "구매 시 주의사항",
      "level": "safe|caution|risk"
    }
  ],
  "chronicDefects": [
    {
      "title": "고질병",
      "detail": "확인 이유",
      "level": "safe|caution|risk"
    }
  ],
  "marketFactors": [
    {
      "title": "시장 변수",
      "detail": "같은 제품군 중고가가 변하는 시장 요인",
      "level": "safe|caution|risk"
    }
  ],
  "purchaseChecklist": [
    {
      "title": "거래 전 확인",
      "detail": "판매자에게 바로 확인할 질문이나 요청",
      "level": "safe|caution|risk"
    }
  ],
  "verdict": "추천/주의/비추천 요약"
}
